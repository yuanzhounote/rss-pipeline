---

## 1. 基础设施先搭起来

按需求里提到的组件，先开通/配置：

| 组件 | 用途 | 最小可用版配置 |
|---|---|---|
| **Cloudflare Worker** | 接收 webhook、爬取、生成 RSS | 1 个 Worker 项目即可，用 `wrangler` 本地开发 |
| **Supabase** | 文章元数据 + 正文存储 | 1 张 `articles` 表，启用 Row Level Security |
| **Cloudflare R2** | 图片持久化 | 1 个 bucket，给 Worker 配好 R2 绑定 |
| **Telegram/飞书机器人** | 用户入口 | 先只做一个（建议 Telegram 更轻），拿到 token 和 webhook URL |

> 建议先用 **Telegram Bot** 跑通，因为 webhook 是 HTTPS POST，调试方便；飞书机器人可以第二阶段再加。

## 2. 数据库表设计

核心表 `articles`：

```sql
create table articles (
  id bigint generated always as identity primary key,
  title text,
  content text,          -- 纯文本或 markdown
  markdown text,
  html text,
  source_url text not null,
  source_type text,      -- 例如 'wechat', 'zhihu', 'generic'
  cover_image text,
  author text,
  status text not null default 'pending',  -- pending / ready / failed
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
```

最小可用版先不做过分结构化，先保证 `status = 'ready'` 时才进 RSS。

---

## 3. Worker 端点设计

一个 Worker 里暴露 3 个路由：

1. **POST `/webhook/telegram`**  
   - 接收 Telegram 发来的消息
   - 提取消息里的 URL
   - 把 URL 写入 `articles`，status = `pending`
   - 返回 “已开始处理”

2. **POST `/extract/:id`** 或内部触发  
   - 读取 `pending` 的文章
   - 判断来源：有 API 走 API（如微信文章可用某些解析服务），没有就开无头浏览器
   - 提取正文、作者、标题、封面图
   - 图片下载到 R2，替换原链接为 R2 公链
   - 更新文章 status = `ready`

3. **GET `/rss.xml`**  
   - 查询 `status = 'ready'` 的文章
   - 按 `created_at` 倒序生成 RSS 2.0 XML
   - 阅读器订阅这个地址即可

---

## 4. 正文抓取策略

这是最容易踩坑的部分，建议分阶段：

### 第一阶段：最小可用版
- 先用一个通用方案：Mozilla 的 `readability`（Node 版本）+ `cheerio`/`linkedom`
- 对普通网页效果足够
- 图片用正则/Cheerio 提取所有 `<img>`，上传到 R2，替换 `src`

### 第二阶段：特殊来源优化
- 微信公众号、知乎、B 站专栏等，做专用解析器
- 需要无头浏览器时，可以用：
  - **Puppeteer on Cloudflare Workers**（有 Workers Browser Rendering 服务）
  - 或单独一个轻量 VPS 跑浏览器，通过 HTTP 接口被 Worker 调用

---

## 5. 图片处理

流程：

```
网页里的图片 URL
    ↓
Worker 下载（注意防盗链，需带 Referer/User-Agent）
    ↓
上传 R2
    ↓
把 article 内容里的图片 URL 替换为 R2 地址
```

> R2 默认可以走 Cloudflare CDN，成本低；如果文章图片多，注意控制并发和 R2 请求费用。

## 6. 部署与调试

建议用 Wrangler 本地开发：

```bash
npx wrangler dev
```

配置 `wrangler.toml`：

```toml
name = "rss-pipeline"
main = "src/index.ts"
compatibility_date = "2026-06-27"

[[r2_buckets]]
binding = "IMAGES"
bucket_name = "rss-images"

[[env_vars]]
SUPABASE_URL = "..."
SUPABASE_KEY = "..."
TELEGRAM_BOT_TOKEN = "..."
```

调试链路：
1. 本地 `/webhook/telegram` 用 ngrok/cloudflare tunnel 暴露
2. Telegram 设置 webhook
3. 发一个链接，看能否写入 Supabase
4. 手动触发 `/extract/:id`
5. 订阅 `/rss.xml` 验证输出

## 7. 迭代路线

| 阶段 | 目标 |
|---|---|
| **MVP** | Telegram 发链接 → Worker 提取正文 → 存 Supabase + R2 → 生成 RSS |
| **v2** | 加入状态机（pending → extracting → ready → failed），失败重试 |
| **v3** | 支持飞书机器人，多源解析器 |
| **v4** | 接入 ReadYou 或自建阅读器前端，支持分类、搜索、全文阅读 |

---

## 8. 风险点提前考虑

- **抓取被封**：注意请求频率、User-Agent、必要时加代理或浏览器渲染
- **图片防盗链**：下载图片时模拟浏览器 headers
- **RSS 输出规范**：`<guid>`、`<pubDate>`、`<content:encoded>` 要完整，否则阅读器显示异常
- **Worker 超时**：复杂网页提取耗时久，可分两步：webhook 先入库，再用 Queue 或 Cron 异步处理
